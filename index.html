台灣產業勞動診斷報告

台灣產業勞動結構
量化診斷

在就業市場中，求職者往往僅能透過「職稱」與「薪資」想像工作環境，卻難以看透產業的「勞動骨架」。本專案透過 Python 交叉分析過去 10 年間台灣 38 個主要產業別的官方數據，以 OS、SR、LR 三大核心指標揭示各產業勞動結構，協助求職者進行「結構性避險」。

01

「就業」與「受僱」的定義Employment vs. Hired

就業 Employment

廣義勞動參與

定義：指個人投入社會生產活動，以獲取報酬或經營家屬事業之狀態。只要從事有酬工作（或每週工作 15 小時以上之無酬家屬工作）皆視為就業。

  • 雇主：自行經營事業並僱用他人者
  • 自營作業者：獨自經營（如個人工作室、自由接案者）
  • 受僱者：受領薪資並受他人指揮者
  • 無酬家屬工作者：幫家屬經營且達到法定時數者
受僱 Employed

狹義薪資勞動

定義：指受雇主僱用，在他人指揮監督下從事工作，並藉此獲取工資或報酬之行為。

本報告後續運算中，「受僱人數」特指此狹義範疇——即正式領薪的受僱勞工，不含自營者與雇主本身。

資料來源：「經濟部中小及新創企業署」政府開放資料平台「年度中小企業之行業與各項指標值及比率」（2013–2024年）↗

02

核心分析指標說明Core Indicators

📌本模型以「產業別」為變數，套入以下核心指標公式。本報告後續提及之所有受僱人數、企業家數、就業人數等數值，均採 2013–2024 十年期間之算術平均值進行運算。
OS
體系穩定度
平均組織規模
Average Organization Scale
OS = 受僱人數 (人)
     ÷ 企業家數 (家)

反映每家企業平均容納多少員工，數值越大代表產業愈集中於大型體系，制度化程度越高。

高 OS (>15)大型組織型。法規保障健全，結構穩定。
低 OS (<5)微型碎片型。穩定度低，受雇主個人特質影響大。
SR
僱用純度
自營與非典型就業率
Self-Employment Rate
SR = (就業人數 − 受僱人數)
     ÷ 就業人數 × 100%

判斷家族化或接案自營的濃度，反映產業中非典型勞動佔比。

高 SR (>20%)高自營/家族化。福利保障可能不完善。
低 SR (<10%)高純度僱用。勞務關係單純明確。
LR
職涯天花板
大企業依賴度
Large Enterprise Reliance
LR = 大企業受僱人數
     ÷ 受僱總人數 × 100%

評估大企業吸納就業資源佔比，反映晉升階梯完整性與職涯天花板高度。

高 LR (>30%)資源集中型。完善晉升與教育訓練體系。
低 LR (<10%)扁平競爭型。管理職缺稀缺，發展受阻。

步驟 3：資料提取與定義運算變數

透過自訂函式 clean_num 清洗帶有千分位逗號或缺漏的字串，再以 groupby('行業別').mean() 算出各產業十年平均，產出四大運算變數：avg_total_employment、avg_total_employed、avg_large_corp_employed、avg_company_count。

In [4]Python
# 自定義函式：清洗帶逗號或缺漏的欄位，轉為純數字
def clean_num(series):
    return pd.to_numeric(series.astype(str).str.replace(',', ''), errors='coerce')

# 1. 總就業人數 (十年期平均)
avg_total_employment    = clean_num(df_employment['全部企業(千人)']).groupby(df_employment['行業別']).mean()

# 2. 總受僱人數 (十年期平均)
avg_total_employed      = clean_num(df_employed['全部企業(千人)']).groupby(df_employed['行業別']).mean()

# 3. 大企業受僱人數 (十年期平均)
avg_large_corp_employed = clean_num(df_employed['大企業(千人)']).groupby(df_employed['行業別']).mean()

# 4. 企業家數 (十年期平均)
avg_company_count       = clean_num(df_company['全部企業(家)']).groupby(df_company['行業別']).mean()
步驟 4：指標運算

以四個變數代入公式，計算 OS、SR、LR，整合為 df_indicators 分析總表。

In [5]Python
# 1. 非受雇人數
avg_non_employed = avg_total_employment - avg_total_employed

# 2. OS — 受僱資料為「千人」，需先 ×1000 還原
avg_os = (avg_total_employed * 1000) / avg_company_count

# 3. SR (%)
avg_sr = (avg_non_employed / avg_total_employment) * 100

# 4. LR (%)
avg_lr = (avg_large_corp_employed / avg_total_employed) * 100

# 5. 整合為分析總表
df_indicators = pd.DataFrame({
    'OS': avg_os, 'SR': avg_sr, 'LR': avg_lr,
}).reset_index().round(2)
步驟 5：指標篩選與排序

透過 sort_values(ascending=False) 抓出極值產業；以 query() 進行多條件過濾。

In [6]Python
# Top 5 SR — 高自營/家族化
df_top_sr     = df_indicators.sort_values(by='SR', ascending=False).head(5)

# Top 5 LR — 大企業壟斷
df_top_lr     = df_indicators.sort_values(by='LR', ascending=False).head(5)

# 穩定大型體系 (OS > 15 且 LR > 30)
df_big_system = df_indicators.query('OS > 15 & LR > 30')

# 扁平化競爭 (OS < 5 且 LR < 10)
df_flat       = df_indicators.query('OS < 5 & LR < 10')
